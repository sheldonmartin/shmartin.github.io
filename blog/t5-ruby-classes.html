Ruby Classes

How to design a class

2015-04-19

Classes in Ruby are used to model real life objects. You’ll start to see the need to use classes as your Ruby programs get more complicated. For instance, you can model a dog using a ruby class. It’s a good example to demonstrate a class, because just about everyone knows what a dog is, and what it can do. However, we’re not going to do that, because dogs are boring. But you know what’s similar to a dog, and a lot more interesting? A wolf :-) Why? Because wolves live in the wild and they hunt, and they’re damn good at it (for reference watch “The Grey” starring Liam Neeson). So lets use that instead.

We need to keep track of the specific attributes of a wolf. Those attributes will be our instance variables. We’ll need to keep track of the wolf’s breed, age, and name (not sure if wolves have names, but lets give them one anyway). These are good examples of instance variables, because they are identifying characteristics of a specific wolf. We might also want to know which pack the wolf belongs to, so lets create a variable for that as well.

name, age, breed, pack

The next thing we need, are instance methods. These methods will describe the types of actions that the wolf class will preform. Wolves howl, there are a couple reasons why, but one of them is for calling other members of their pack. They also need to hunt for their own food, since they live in the wild. Once they get that food, they need to eat it. So we now have three actions that wolves need to preform.

howl, hunt, eat

Now what if we wanted to know how many instances of the wolf class exist at a given time. In order to do that we’d need a class variable to keep track of the number of wolves. We’d also need a class method that increments the wolf counter each time a new wolf object is created.

population, increment_population

One way to know when a wolf is created, is when it’s initialize method is called. So we’ll create an initialize method that calls the increment_population method. So our wolf class now looks like this:

Wolf Class

Instance Variables: name, age, breed, pack

Instance Methods: initialize, howl, hunt, eat

Class Variable: population

Class Method: increment_population

You can use a similar train of thought to create classes for other types of objects. Just keep in mind the characteristics of the object (variables), and the actions it will perform (methods).
