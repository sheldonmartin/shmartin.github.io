Arrays and Hashes

When and how to use them

2015-04-04

Arrays

An array is pretty much a list of objects. Think of a shopping list. You put items on it that you want to remember to buy at the grocery store. Arrays in Ruby serve a similar purpose, they are there to store a list of related items. If you wanted to create an empty array in Ruby, you'd use the following code:

shopping_list = []

To add items to that list, you'd use either one of the following lines of code:

shopping_list << 'apple' shopping_list.push('orange')

Each item in a an array can be accessed using its index. Array indexes start at 0, and can be used by putting square brackets next to the array name, with the index number in between. The following code will print out the first item we put in the array:

puts shopping_list[0] => apple

This code will get you the second item in the array:

puts shopping_list[1] => orange

In order to change the value of an array, you'd use the following code:

shopping_list[0] = 'apple juice' puts shopping_list[0] => apple juice

Hashes

Hashes are a lot like arrays, they are list of related objects. The major difference between an array and a hash, is in the way hashes store their objects. In a hash, objects are stored as key-value pairs. The key is used to identify a specific object in a hash, similar to arrays. But keys in hashes don't have to be numbers, they can also be strings. Lets take that same grocery list. What if you wanted to not only store the items, but the amount of each item you needed to buy. To create an empty hash, you'd use the following code:

shopping_list = {}

Now to assign items to that list, you'd do this:

shopping_list['apple'] = 5 shopping_list['orange'] = 3

If you want to see the value assigned to a key. You'd use syntax similar to an array, but instead of using the index, you use the key. In our example, this would get you the amount of the item you need to buy.

puts shopping_list['apple'] => 5

If you wanted to see if a hash has a certain key (perhaps you forgot if you put it on your list or not). You can use the "has_key?" method to check.

puts shopping_list.has_key?('banana') => false

If you need to reference a list by using a string instead of an integer, or if speed is a concern (hashes tend to have faster performance than arrays), you're better off using a hash. But the simplicity of arrays, make them a good choice when first starting to use data structures.
