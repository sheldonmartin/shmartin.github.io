Inheritance vs Composition

2015-04-23

Inheritance

Inheritance is when you have one class that is based on another class. For instance, if you created a class to represent an animal, you might then decide to create a dog class that is derived from the animal class. In Ruby, you'd do this by using the following code:

def Animal def eat puts "Animal eats" end
def sleep puts "Animal sleeps" end end
def Dog < Animal def bark puts "Dog Barks" end end

The two classes now have an parent-child relationship. This is also known as an "is-a" relationship, Dog is-a type of Animal. The Dog class gains the methods defined in the Animal class, it does not need to define them on its own. It can also choose to override the methods in the Animal class, choosing its own implementation for those methods. Any new methods defined in Dog belong to it and any classes that may derive from it in the future. In Ruby, each child can only have one parent, but there are other languages that allow more than one parent, this is called multiple inheritance.

Composition

Composition is when you use instance variables that are actually other objects. So instead of inheriting methods from another class, you can simply create an instance of that class and call the methods that you need. You can more or less the same benefits of inheritance, with out having a parent-child relationship.

def Animal def eat puts "Animal eats" end
def sleep puts "Animal sleeps" end end
def Dog < Animal @animal = Animal.new
def eat @animal.eat end
def sleep @animal.sleep end
def bark puts "Dog Barks" end end

Which one to use?

With composition you have to write code to define the methods you want to use from another class. This means you need to write additional code, that wouldn't be necessary if you used inheritance. However, with inheritance you can't pick and choose which methods you want implemented, you have to take them all and override the ones you want changed. If you only want some methods (or maybe just a few) implemented from another class, you are probably better off using composition.
