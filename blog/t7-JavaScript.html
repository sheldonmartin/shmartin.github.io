Ruby Classes vs JavaScript Constructors

2015-04-02

Ruby is a class based object oriented language. This means you can create classes which act as blueprints for your objects. Although you don't have to use a class to create your own object, you are better off doing so if you need to create more than one of the same object. A simple class might look like this:

class MyClass @@var_1 = 1 def initialize(var_2) @var_2 = var_2 end end

At the top of the class I've placed a class variable (you know it's a class variable because it has a double @), followed by an initialize method, which is run when an instance of the class is created. Inside the initialize method is an instance variable. A separate version of the instance variable is created for each instance of a class. However, a class variable belongs to the class itself, and only one version of it exist. To create an instance of a class, you assign a variable to the class name and call it's "new" method.

class MyClass @@var_1 = 1 def initialize(var_2) @var_2 = var_2 end end
object_1 = MyClass.new(1) object_2 = MyClass.new(2)

In JavaScript there are no classes, there are only objects. To create objects in a way similar to a Ruby class, you use a constructor. A simple JavaScript constructor might look like this:

function myClass(var1, var2) { this.var1 = var1; this.var2 = var2; }

Once you create a JavaScript constructor, you can then create instances of it, similar to how you'd create instances of a class in Ruby:

function myClass(var1, var2) { this.var1 = var1; this.var2 = var2; }
var object1 = new myClass(1,2); var object2 = new myClass(3,4);

Because JavaScript doesn't have any classes there are no class variables either, only instance variables. These differences make JavaScript easier in some ways because of it's simplicity. However, this comes at a cost. Ruby has built-in encapsulation, variables are hidden from the outside by default, and only accessible through method calls. This is not the same in JavaScript, object properties can be accessed freely from outside the class. To get encapsulation in JavaScript, you'd have to put in more work since, it's not built into the language. So far in my experience, it seems writing good JavaScript is harder than writing good Ruby. However, I'm still learning both so in time we'll see if this holds up.
